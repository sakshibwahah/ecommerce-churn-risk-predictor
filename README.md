> XGBoost churn risk prediction · SHAP interpretability · RFM segmentation · Folium geospatial · Streamlit dashboard

## Model Design Rationale

### Why not predict "no repeat purchase"?

The Olist dataset uses **anonymised per-transaction customer IDs** — a single physical customer generates a new `customer_id` for every order. As a result:

- Only ~3% of distinct `customer_unique_id` values appear in more than one order.
- Defining churn as "no repeat purchase in the next N months" produces a **100% churn rate** for any reasonable window.
- Defining it as "no purchase within 180 days" and including `recency_days` as a feature causes **AUC = 1.0** (direct label leakage — the feature encodes the label boundary).

### What the model actually predicts

This project instead predicts **customer churn risk** — the probability that an order results in a **dissatisfied experience** (review score 1 or 2). This framing:

- **Eliminates all leakage**: every feature (delivery delay, freight ratio, product quality signals, geography) is observable before the review is submitted.
- **Is directly actionable**: improving delivery speed and reducing freight cost measurably lowers churn risk.
- **Is measurable in production**: any new order can be scored in real time from fulfillment data alone.

| Feature | Why it predicts dissatisfaction |
|---------|--------------------------------|
| `delivery_delay_days` | Primary driver — customers penalise late delivery heavily |
| `actual_delivery_days` | Absolute time perception, independent of estimate |
| `price` / `freight_ratio` | Disproportionate shipping cost creates frustration |
| `product_photos_qty` | Low-quality listings lead to expectation mismatch |
| `description_length` | Short descriptions correlate with information asymmetry |
| `same_state` | Cross-state logistics are slower and more likely to be late |
| `customer_state_enc` | Regional delivery infrastructure varies significantly |
| `category_enc` | Certain categories (electronics, large furniture) have higher dispute rates |

## Quick Start

### 1. Set Up Environment
```bash
python -m venv venv
source venv/bin/activate  # On Windows use: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Ingest Data into SQLite
```bash
python data_preprocessing.py
```

### 3. Run Notebooks (optional — for EDA and RFM)
```bash
jupyter notebook notebooks/01_sql_eda.ipynb
# Repeat for 02, 03, 04, 05
```
Run in order: 01 → 02 → 03 → 04 → 05.

### 4. Launch Streamlit Dashboard
```bash
streamlit run app/streamlit_app.py
```
Opens at `http://localhost:8501`

## Model Performance

| Metric | Value |
|--------|-------|
| Algorithm | XGBoost (GBM with L1/L2 regularisation) |
| Target | Dissatisfied experience (review score 1 or 2) |
| CV AUC (5-fold) | 0.836 ± 0.003 |
| Train AUC | 0.860 |
| At-risk base rate | 12.8% |
| Explainability | SHAP TreeExplainer |
| Class imbalance | `scale_pos_weight` (6.8×) |
